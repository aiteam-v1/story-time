# PRD: Kids Story Reading App\n\n## 1. Overview\n**Product Name:** StoryTime (Working Title)\n**Platform:** React Native (iOS & Android)\n**Target Audience:** Parents reading to children (ages 3-7) and early readers.\n\n**Core Value:** A simple, distraction-free reading experience that makes the child the hero of the story by dynamically replacing character names.\n\n## 2. Problem Statement\nKids engage more deeply when they see themselves in the story. Customizing physical books is expensive and slow. Existing apps are often cluttered with gamification that distracts from the reading experience.\n\n## 3. User Flow\n1.  **Home Screen:**\n    *   User sees a vertical list of available stories.\n    *   Each item shows a thumbnail and title.\n    *   *Constraint:* Stories are hardcoded in the app for v1 (no backend).\n\n2.  **Story Selection:**\n    *   User taps a story.\n    *   **Popup/Modal appears:**\n        *   Text: "This story is about [Default Name]. Who should be the hero?"\n        *   Input Field: [ Enter Name ] (Default is empty or placeholder).\n        *   Action Buttons: "Start Reading" (uses input name) | "Cancel".\n        *   *Logic:* If input is empty, use the default story character name.\n\n3.  **Reading Experience:**\n    *   **Layout:** Split screen. Top 50% = Illustration. Bottom 50% = Text (3-4 sentences).\n    *   **Name Replacement:** Every instance of the [Default Name] in the text is replaced by the [User Input Name].\n    *   **Audio:** Text-to-Speech (TTS) engine reads the text aloud immediately upon loading the page.\n    *   **Navigation:**\n        *   Big "Next" arrow to advance.\n        *   "Back" arrow to return to previous page.\n        *   "Home" icon (top corner) to exit.\n\n4.  **End of Story:**\n    *   Final illustration.\n    *   "Read Again" or "Back to Stories" buttons.\n\n## 4. Functional Requirements\n\n### FR1: Story Data Structure (JSON)\nStories must be stored locally in a JSON format containing:\n*   uid=1000(mha) gid=1000(mha) groups=1000(mha),27(sudo),100(users): unique string\n*   : string\n*   : string (the name to find and replace)\n*   : array of objects\n    *   : local asset path\n    *   : string (sentences)\n\n### FR2: Name Injection\n*   The app must perform a case-insensitive find-and-replace on the story text before rendering.\n*   The replaced name must match the casing of the surrounding sentence (Capitalized at start, etc. - for v1 simple string replacement is acceptable if we assume names are always capitalized).\n\n### FR3: Audio (TTS)\n*   Use native platform TTS (Expo Speech or React Native TTS).\n*   Must read the *modified* text (with the child's name).\n*   Audio starts automatically when page loads.\n*   Stop audio if user navigates away/forward.\n\n## 5. Non-Functional Requirements\n*   **Performance:** Instant page turns.\n*   **Offline:** App must work 100% offline.\n*   **UI/UX:** Large buttons, kid-friendly fonts, high contrast.\n\n## 6. Out of Scope (v1)\n*   Backend/API fetching (stories are local).\n*   User accounts/login.\n*   Saving custom names for next time.\n*   Voice recording (custom narration).\n*   Complex animations.\n*   In-app purchases.\n\n## 7. Success Metrics\n*   Successful story completion (reaching the last page).\n*   Re-read rate (opening the same story again).\n
